# 計画スキル

`/plan <タスク概要>` で起動。引数がなければユーザーに確認する。

## 手順

### 1. 要件整理

タスクの概要を整理し、以下を明確にする。

- 何を実現するか（目的）
- 影響するファイル・機能・レイヤー
- 考慮すべき制約（認証・バリデーション・既存APIとの整合性など）
- `docs/` 以下のドキュメントに未決定事項があればオーナーに確認する

必要なら `codebase-research` スキルを呼び出してコードベースを調査する。

### 2. TODO.md への書き出し（必須）

以下のフォーマットで `/Users/naary/repo/muu-cms/TODO.md` を**必ず Write ツールで作成・上書き**する。
ターミナルへのマークダウン出力だけで済ませてはいけない。

```markdown
# TODO: <タスク名>

## 目的
<1〜3行で目的を記述>

## 作業ステップ

- [ ] <ステップ1>（テスト作成）
- [ ] <ステップ2>（実装）
- [ ] ...

## 影響範囲
<変更するファイル・レイヤーの一覧>

## 注意事項
<制約・考慮点があれば記述>
```

### 3. ユーザーへの提示と承認待ち

TODO.md の内容をターミナルにも表示し、以下の文言で承認を求める。

> TODO.md に計画を書き出しました。内容を確認して「承認」または修正指示をお願いします。承認後にフェーズ2（実装）へ進みます。

**承認が得られるまで実装（ファイル編集・コード生成）を開始しない。**

修正指示が来た場合は TODO.md を更新して再度提示する。

## 制約

- TDD で進める前提でステップを組む（テスト作成 → 実装の順）
- EloquentとLaravelのmigrationのみ。Raw SQLを書かない
- ドキュメントに記載のない仕様を勝手に実装しない
- リファクタリングは依頼されていない限りステップに含めない
