# API設計ドキュメント

> **最終更新：** 2026-04-05

---

## 概要

APIは2種類に分かれる。

| 種別 | 説明 | 認証 |
|------|------|------|
| 管理API | 管理画面（Inertia.js）からのハコ・モノ・コンテンツ操作 | 必要（パスキー認証） |
| モノAPI | 外部から利用するコンテンツ取得エンドポイント | 不要 |

---

## 共通仕様

### レスポンス形式

- Content-Type: `application/json`
- 文字コード: UTF-8

### エラーレスポンス

```json
{
  "message": "エラーの説明"
}
```

| HTTPステータス | 意味 |
|--------------|------|
| 400 | リクエストの形式が不正 |
| 401 | 未認証 |
| 403 | 権限なし |
| 404 | リソースが存在しない |
| 422 | バリデーションエラー |
| 500 | サーバーエラー |

---

## モノAPI

### GET `/api/v1/{slug}`

外部から利用するコンテンツ取得エンドポイント。認証不要。

**パスパラメータ**

| パラメータ | 説明 |
|-----------|------|
| `slug` | モノのスラッグ（`blueprints.slug`） |

---

#### タイプ `ひとつ` のレスポンス

コンテンツのデータをそのままJSONオブジェクトで返す。コンテンツが未登録の場合は404を返す。

```json
{
  "title": "利用規約",
  "body": "本文テキスト..."
}
```

---

#### タイプ `たくさん` のレスポンス

```json
{
  "contents": [
    {
      "title": "記事タイトル",
      "body": "本文テキスト..."
    }
  ],
  "totalCount": 42
}
```

**クエリパラメータ**

| パラメータ | 型 | デフォルト | 説明 |
|-----------|----|-----------|------|
| `limit` | integer | 10 | 取得件数（最大100） |
| `offset` | integer | 0 | 取得開始位置 |

---

#### レスポンスのキー

`contents.data` に格納されているキーのうち、**現在のスペックに存在するパラメータの `name` のみ**を返す。削除済みパラメータのキーはレスポンスから除外する。

---

## 管理API

管理画面はInertia.js（サーバーサイドレンダリング）で実装する。管理APIはLaravelのルーティングで定義し、JSON APIとして切り出さない。

以下はInertiaルートとして実装するリソース一覧。

### ハコ

| 操作 | 説明 |
|------|------|
| 一覧取得 | 自分が所属するハコの一覧 |
| 作成 | ハコを新規作成 |
| 更新 | ハコの名前を変更 |
| 削除 | ハコと配下のモノ・コンテンツをすべて削除 |

### モノ

| 操作 | 説明 |
|------|------|
| 一覧取得 | ハコに属するモノの一覧 |
| 作成 | モノを新規作成。同時にスペックを1件作成する |
| 更新 | モノの名前を変更（`slug` と `type` は変更不可） |
| 削除 | モノと配下のコンテンツをすべて削除 |

### スペック・パラメータ

| 操作 | 説明 |
|------|------|
| 取得 | モノのスペック（パラメータ一覧）を取得 |
| パラメータ追加 | スペックにパラメータを追加 |
| パラメータ削除 | スペックからパラメータを削除 |
| パラメータ順序変更 | `sort_order` を更新 |

> **注意：** パラメータの `name` は変更不可。変更しようとした場合はエラーを返す。

### コンテンツ

| 操作 | 説明 |
|------|------|
| 一覧取得 | モノに属するコンテンツの一覧 |
| 取得 | コンテンツ1件の取得 |
| 作成 | コンテンツを新規作成。`type = 'single'` の場合は2件目の作成を禁止する |
| 更新 | コンテンツのデータを更新 |
| 削除 | コンテンツを削除 |

### ユーザー・認証

| 操作 | 説明 |
|------|------|
| 初回セットアップ | DBにユーザーが0件のときのみ使用可能。管理者ユーザーを作成する |
| パスキー登録 | パスキーをユーザーに紐づける |
| パスキーログイン | パスキーで認証する |
| リカバリ情報登録 | メールアドレスとパスワードを登録する |
| リカバリログイン | メールアドレスとパスワードで認証する |
| ログアウト | セッションを破棄する |
