# 要件定義書

---

## プロダクト概要

セルフホスティング型ヘッドレスCMS。ユーザーがスキーマを定義し、コンテンツをGET APIとして公開できる。OSSとしてレンタルサーバーからPaaS環境まで幅広く動作することを目標とする。

---

## 用語定義

| 用語 | 意味 |
|------|------|
| ルーム | 最大の管理単位。モノの集合 |
| モノ | ユーザーが定義するコンテンツの型。スペックとAPIエンドポイントを持つ |
| スペック | モノのスキーマ定義。パラメータの集合 |
| パラメータ | スペックを構成するフィールドの1つ |
| コンテンツ | モノに対して投稿される実データの1件 |

---

## ユーザー

### 初期セットアップ

- DBにユーザーが0件の状態でアクセスした場合、管理者ユーザーの作成画面に遷移する
- 管理者ユーザーを1人作成することでシステムが利用可能になる

### ロール

| ロール | 権限 |
|--------|------|
| 管理者 | ルームの作成・管理、すべての操作が可能 |

> **備考：** MVPはシングルユーザー（管理者のみ）を前提とする。将来の複数ユーザー対応に備え、DB設計は拡張可能な構造にする。

### 認証

- **プライマリ：** パスキー認証
- **リカバリ：** メールアドレスとパスワード（初回登録時は任意）

パスワードに関する注意事項をシステムが表示する：
- 人間が推測・記憶できないパスワードであること
- Apple パスワードアプリや Chrome パスワードマネージャなど、専用ツールで管理すること

## ルーム

- 管理者ユーザーはルームを作成できる
- ルームはモノを持つ

---

## モノ

### 属性

| 属性 | 説明 | 制約 |
|------|------|------|
| モノID | APIエンドポイントの識別子 | 半角英数スラッグ、ルーム内でユニーク |
| 名前 | 管理画面上の表示名 | 日本語可 |
| タイプ | `ひとつ` または `たくさん` | 変更不可 |

### タイプの意味

| タイプ | ユースケース例 | コンテンツ件数 |
|--------|--------------|--------------|
| `ひとつ` | プロフィール、利用規約、プライバシーポリシー | 常に1件 |
| `たくさん` | ブログ記事、ポートフォリオ実績 | 複数件 |

---

## スペックとパラメータ

- モノは必ず1つのスペックを持つ
- スペックは1つ以上のパラメータを持つ

### パラメータの属性

| 属性 | 説明 | 制約 |
|------|------|------|
| パラメータ名 | APIレスポンスのキー名 | 半角英数 |
| 説明 | 管理画面上の補足テキスト | 文字列 |
| 型 | `string`（MVPは文字列のみ） | - |
| 必須 | 入力を必須とするか | デフォルト：必須 |
| 文字数制限 | 最大文字数 | 任意 |

---

## モノAPI（公開エンドポイント）

- 認証不要のGETエンドポイント
- エンドポイント：`{オリジン}/api/v1/{モノID}`

### レスポンス形式

**タイプ：`ひとつ`**

コンテンツのJSONオブジェクトをそのまま返す。

**タイプ：`たくさん`**

```json
{
  "contents": [],
  "totalCount": 0
}
```

## MVPスコープ

### 含むもの

- 管理者ユーザーの初回作成
- パスキー認証（メールアドレス・パスワードによるリカバリ含む）
- ルームのCRUD
- モノのCRUD
- スペック・パラメータの定義
- コンテンツのCRUD
- モノAPI（GET）

### 含まないもの

- 複数ユーザー・権限管理
- メディア管理（画像アップロード等）
- モノAPIへの書き込み（POST/PUT/DELETE）
- コンテンツの公開ステータス管理
- パラメータの型（string以外）
