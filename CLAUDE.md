# CLAUDE.md

このドキュメントはClaude Codeが作業を進めるための指針。実装中に判断に迷ったときはここに立ち返ること。

---

## ドキュメント構成

| ファイル | 内容 |
|----------|------|
| `docs/requirements.md` | 要件定義。何を作るかの全体像 |
| `docs/design/database.md` | DB設計。テーブル定義・設計原則 |
| `docs/design/api.md` | API設計。エンドポイント・レスポンス仕様 |
| `docs/design/frontend.md` | フロントエンド実装指針。デザイン・Async React |
| `TODO.md` | 現在のタスクリスト（作業開始時に作成・更新する） |

---

## 開発フロー

以下のサイクルを厳守すること。

```
1. TODO.md にタスクを書き出す
2. タスクを1つ選ぶ
3. そのタスクの正常系テストを1つだけ書く
4. テストを実行して落ちることを確認する（Red）
5. テストが通るように実装する（Green）
6. 全件テストが通ることを確認する
7. 2に戻る
```

---

## テストの書き方

- テストケースは**日本語**で記述する
- テスト名はそのテストが何を確認するかを1行で表す仕様として書く
- テスト名の例：`ルームを作成できる`、`パラメータのnameは変更できない`

---

## 実装上の制約

### DB

- EloquentとLaravelのmigrationのみで実装する
- Raw SQLを書かない
- DB固有の関数（`JSON_EXTRACT` 等）を使わない
- JSONカラムへのアクセスは `$casts` 経由に統一する

### ドキュメントとの整合性

- 実装内容がドキュメントと食い違う場合は、実装を止めてオーナーに確認する
- ドキュメントに記載のない仕様を勝手に実装しない
- ドキュメントを更新した場合は更新日を修正する

### 未決定事項

`docs/` 以下のドキュメントに未決定と明記されている事項は、実装着手前にオーナーに確認すること。

---

## TODO.md の運用

作業開始時に `TODO.md` を作成し、タスクをリストアップする。完了したタスクはチェックを入れる。タスクが増えた場合はその都度追記する。

```markdown
# TODO

- [x] 完了したタスク
- [ ] 未着手のタスク
```

---

## レビュー担当

実装・テストはClaude Codeが行う。オーナーはドキュメントレビューと動作確認のみを担当する。
